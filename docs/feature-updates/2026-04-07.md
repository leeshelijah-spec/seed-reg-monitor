# 2026-04-07

## Git Bash 실행 경로 정리

- `start_dashboard.sh`, `scripts/start_dashboard.sh`를 추가해 PowerShell 대신 Git Bash 기준으로 대시보드를 실행하도록 정리
- `start_dashboard.cmd`가 Git Bash를 직접 호출하도록 변경하고 `--port` 같은 인자를 그대로 전달하도록 보강
- Git Bash 환경에서 `coproc` 배열 변수 문제로 중단되던 구간을 FIFO 기반 로그 읽기로 교체해 실행 안정성을 높임

## 동기화 진행도와 팝업 개선

- 상단 `규제 동기화`, `뉴스 수집` 버튼이 비동기 API를 호출하도록 정리
- `Sync Status`는 상시 패널이 아니라 동기화 버튼을 눌렀을 때만 열리는 팝업으로 변경
- startup sync 콘솔 출력은 여러 줄 로그 대신 한 줄 게이지가 갱신되는 형태로 다듬음
- 규제/뉴스 동기화 진행 상태를 백그라운드에서 관리하는 `sync_progress` 서비스를 추가

## Startup Sync 제어

- 서버 시작 직후 실행되는 startup sync에 최근 성공 이력 기반 skip 정책을 추가
- 규제는 최근 24시간 내 성공 이력이 있으면 자동 동기화를 건너뜀
- 뉴스는 최근 3시간 내 성공 이력이 있으면 자동 수집을 건너뜀
- skip 시에도 콘솔 게이지에 건너뛴 이유가 표시되도록 정리

## 규제 수집 범위 확장

- 규제 동기화 기본 조회 범위를 `REGULATION_SYNC_LOOKBACK_DAYS` 설정값으로 통일하고 기본값을 14일로 확대
- 법령 이력 조회 건수를 500건, 행정규칙 검색 건수를 100건으로 확장
- 첫 조회 결과가 0건이면 최소 30일까지 범위를 넓혀 한 번 더 재시도하는 fallback 규칙을 추가

## 뉴스 분석 보강

- 스포츠성 기사 신호가 강하고 종자·검역·병해충 같은 도메인 신호가 약한 경우 자동으로 `잡음` 처리
- 스포츠 잡음은 `review_status = 잡음`, `relevance_score = 0`, `analysis_trace.sports_noise = true`로 기록
- 뉴스 KPI와 Top 5 요약이 피드백 결과를 더 직접 반영하도록 대시보드 집계 규칙을 정리
