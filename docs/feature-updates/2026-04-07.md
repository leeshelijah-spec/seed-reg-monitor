# 2026-04-07

## Git Bash 실행 경로 정리

- `start_dashboard.sh`, `scripts/start_dashboard.sh`를 추가해 PowerShell 대신 Git Bash 기준으로 대시보드를 실행하도록 정리
- `start_dashboard.cmd`도 Git Bash를 직접 호출하도록 변경하고, `--port` 같은 인자를 그대로 전달하도록 보강
- Git Bash 환경에서 `coproc` 배열 변수로 종료되던 문제를 FIFO 기반 로그 읽기로 교체해 실행 안정성을 높임

## 동기화 진행도 UI 추가

- 상단 `규제 동기화`, `뉴스 수집` 버튼을 비동기 API 호출 방식으로 전환
- 규제/뉴스 동기화 진행 상태를 백그라운드에서 관리하는 `sync_progress` 서비스를 추가
- `Sync Status`는 항상 노출하지 않고, 동기화 버튼을 눌렀을 때만 팝업으로 열려 진행도 게이지를 표시하도록 변경
- 규제 동기화는 수집/분류/저장/알림 단계 기준으로, 뉴스 수집은 키워드별 진행 기준으로 퍼센트가 갱신되도록 연결

## Startup Sync 동작 개선

- 서버 시작 후 자동 실행되는 startup sync 콘솔 출력 방식을 한 줄 게이지 형태로 변경
- startup sync는 매번 실행하지 않도록 정책을 추가
- 규제는 최근 24시간 내 성공 이력이 있으면 skip
- 뉴스는 최근 3시간 내 성공 이력이 있으면 skip
- skip 시에도 콘솔 게이지에는 건너뛴 이유가 표시되도록 정리

## 규제 수집 범위 확대

- 규제 자동/수동 동기화 기본 조회 범위를 설정값 `REGULATION_SYNC_LOOKBACK_DAYS`로 통일하고 기본값을 14일로 확대
- 법령 이력 조회 건수를 500건으로, 행정규칙 검색 건수를 100건으로 확장
- 첫 조회 창에서 결과가 0건이면 더 넓은 창(최소 30일)으로 한 번 더 재시도하도록 fallback 규칙 추가
- 최근 `sync_runs`와 적재 현황 기준으로, 최근 며칠간 규제 수집 성과가 0건이던 상황을 완화하기 위한 방향으로 보강

## 뉴스 잡음 규칙 보강

- 스포츠성 기사(`축구`, `야구`, `골프`, `리그`, `선수`, `경기` 등) 신호가 강하고 종자/검역/병해충 같은 도메인 신호가 약한 경우 자동으로 `잡음` 처리
- 스포츠 잡음은 `review_status = 잡음`, `relevance_score = 0`, `analysis_trace.sports_noise = true`로 기록되도록 추가
- 검색 API 단계는 그대로 두고, 수집 후 분석 단계에서 걸러 운영 키워드의 범용성은 유지

## 대시보드/템플릿 안정화

- `dashboard.html`의 뉴스 그룹 반복 변수 오타(`kroup`)로 메인 화면이 500 오류를 내던 문제를 수정
- `base.html`을 정상 UTF-8 기준으로 다시 정리해 상단 버튼과 동기화 팝업 스크립트가 안정적으로 동작하도록 복구
- 실행 중이던 서버를 새 코드로 재기동해 상단 버튼, 팝업, 동기화 API가 실제 반영되도록 조치
