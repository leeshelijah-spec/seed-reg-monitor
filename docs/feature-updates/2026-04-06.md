## 뉴스 피드백 UX 단순화

- 기사 피드백을 체크 후 저장하는 2단계 대신 `관련`, `잡음` 버튼을 누르면 즉시 반영되는 1단계 흐름으로 변경
- 개별 기사와 묶음 기사 모두 같은 액션 값(`feedback_action`)으로 처리하도록 라우트 해석 로직을 정리
- 버튼 기반 피드백 파싱과 잘못된 액션 입력에 대한 회귀 테스트를 추가

## 뉴스 분석과 피드백 연동 강화

- 피드백 학습에서 구조화된 상태뿐 아니라 코멘트까지 다시 읽어오도록 확장
- 재수집 기사와 수동 피드백 저장 시 권장 대응 문구에 피드백 조치사항을 함께 반영하도록 보강
- `잡음`으로 분류된 기사는 KPI, 추이, 경영 요약 집계에서 제외하고, `관련`으로 확정된 기사는 분석 우선순위와 요약 문구에 우선 반영

## KPI/대시보드 레이아웃 재정비

- 상단 액션 버튼 3개를 동일한 크기로 맞추고 KPI 카드 내부 정렬과 간격을 정리
- 분석 영역은 차트 2개를 세로 스택으로 배치하고 경영 요약 패널을 우측에서 높게 쓰는 구조로 변경
- KPI 패널은 좌측 대표 카드가 세로 3행을 차지하고 우측 요약 카드 3개가 위에서 아래로 쌓이는 구조로 재배치

## 주제/분포 표현 방식 개편

- 뉴스 Top 5는 `관련`으로 확정된 기사만 집계하도록 기준을 조정
- Top 5 항목은 `1. 규제/정책 (16건)`처럼 한 줄 형식으로 압축해 카드 높이를 줄임
- 규제 중요도 분포는 차트 대신 텍스트 요약 카드로 대체하고, 함께 검토 반영 현황도 표시하도록 변경
