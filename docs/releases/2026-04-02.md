# 2026-04-02

## 뉴스 대시보드 UI 및 문서 정리

- 상단 Hero 박스를 제거하고 `Seed Regulation Monitor` 하단에 설명 문구를 작게 배치
- `규제 동기화`, `뉴스 수집` 버튼을 더 작은 가로형 버튼으로 정리
- 규제 모니터링 현황과 뉴스 분석 KPI를 각각 2x2 카드 배치로 재구성
- 뉴스 섹션의 필터와 관리자용 키워드 관리를 상단 `필터` 버튼으로 여는 토글 패널로 통합
- 중요 기사 상위 리스트를 제거하고, 차트와 경영 요약/운영 현황을 1x2 레이아웃으로 정리
- `.env.local` 로드 상태를 확인하고 네이버 API 설정이 실제 화면에 반영되도록 정리
- 업데이트 이력 관리 방식을 `feature-updates`에서 `release notes` 중심으로 전환

## 대시보드 그리드 축소 조정

- 규제 KPI 4개 카드의 높이와 폭을 줄여 더 촘촘한 2x2 그리드로 재배치
- 뉴스 KPI 4개 카드도 동일하게 축소된 2x2 그리드로 재배치
- `카테고리별 기사 건수`, `키워드별 언급 추이`, `경영 요약 패널`, `운영 현황`을 하나의 2x2 그리드로 재구성
- 960px 이하에서도 KPI/차트/운영 패널이 2열을 유지하도록 반응형 규칙을 조정
- 운영 현황 표에 중복 건수를 함께 표시해 뉴스 수집 성공과 신규 저장 여부를 구분할 수 있게 정리

## 미검토 중심 대시보드 재구성

- 규제 KPI와 뉴스 KPI를 최상단으로 올리고 `2x2 + 2x2` 형태의 개요 영역으로 재배치
- `카테고리별 기사 건수`, `키워드별 언급 추이`, `경영 요약 패널`, `운영 현황`이 그 다음 순서의 2x2 패널로 이어지도록 정리
- `최근 등록 목록`을 `미검토 목록`으로 변경하고 기본적으로 조치 이력이 없는 규제만 표시하도록 수정
- `미검토 기사 목록`을 추가해 기본적으로 피드백하지 않은 기사만 표시하고, 규제/기사 목록 모두 `전체목록보기` 토글을 지원
- 기사 피드백 저장 후 페이지 최상단이 아닌 기사 목록 섹션으로 다시 돌아오도록 리다이렉트 위치를 보정
- `분석 및 운영 현황` 4개 패널을 하나의 큰 박스 안에 감싸고, 미검토 보기 상태와 무관하게 누적 집계가 유지되도록 목록 필터와 집계 필터를 분리
